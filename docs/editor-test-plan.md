# 절곡 단면 편집기 테스트 계획

## 1. 테스트 환경

- 개발 서버: `npm run dev`
- 접속 주소: `http://localhost:3000`
- 권장 브라우저: Chrome 또는 Edge 최신 버전
- 데스크톱 기준: 1440 × 900 이상
- 모바일 기준: 390 × 844

테스트를 시작할 때 페이지를 새로고침하여 기본 예제 도면 3개 선과 제품 길이 2,400 mm, 수량 10이 표시되는지 확인한다.

## 2. 자동 테스트 실행

```bash
npm test
npm run lint
npm run build
```

- `npm test`: 계산 엔진, 도메인 모델, 편집 상태와 Undo/Redo를 검증한다.
- `npm run lint`: 코드 규칙과 React Hook 사용 오류를 검사한다.
- `npm run build`: TypeScript 검사와 실제 배포 빌드를 검증한다.

개발 중 테스트를 계속 실행하려면 다음 명령을 사용한다.

```bash
npx vitest
```

### P0: 반드시 통과

| ID | 기능 | 절차 | 기대 결과 |
|---|---|---|---|
| ED-001 | 초기 화면 | 페이지를 새로고침한다 | 선 3개가 모두 보이고 첫 번째 선이 선택된다 |
| ED-002 | 화면 맞춤 | `화면 맞춤`을 누른다 | 전체 도면이 캔버스 중앙에 여백을 두고 표시된다 |
| ED-003 | 연속 선 작성 | 연필 아이콘을 누르고 빈 공간을 2회 클릭한다 | 첫 클릭은 시작점, 두 번째 클릭은 끝점이 되어 선이 생성된다 |
| ED-004 | 기존 도면 연장 | 예제 도면에서 연필 아이콘을 누르고 한 점을 클릭한다 | 마지막 끝점에서 클릭 위치까지 네 번째 선이 연결된다 |
| ED-005 | 수평·수직 스냅 | 이전 끝점과 거의 수평 또는 수직인 위치를 클릭한다 | 선이 정확한 수평 또는 수직으로 보정된다 |
| ED-006 | 선 선택 | 화살표 아이콘을 누르고 다른 선을 클릭한다 | 선택 선이 청록색으로 표시되고 우측 속성이 바뀐다 |
| ED-007 | 관절점 이동 | 선 끝의 흰색 관절점을 드래그한다 | 양쪽 인접 선이 끊어지지 않고 길이가 갱신된다 |
| ED-008 | 숫자 길이 변경 | 첫 선을 선택하고 길이를 100에서 120으로 바꾼다 | 첫 선은 120 mm가 되고 뒤쪽 형상이 이동하며 전개 폭은 250 mm가 된다 |
| ED-009 | Undo | ED-008 후 실행 취소를 누른다 | 길이와 전개 폭이 각각 100 mm, 230 mm로 복원된다 |
| ED-010 | Redo | ED-009 후 다시 실행을 누른다 | 길이 120 mm와 전개 폭 250 mm가 다시 적용된다 |
| ED-011 | 중간 선 삭제 | 두 번째 선을 선택하고 휴지통 아이콘을 누른다 | 선이 삭제되고 앞뒤 선이 연결되며 새 길이가 계산된다 |
| ED-012 | 새 도면 | `새 도면`을 누르고 캔버스를 두 번 클릭한다 | 기존 선이 제거되고 두 클릭 사이에 첫 선이 생성된다 |
| ED-012A | 도형 닫기 | 선을 2개 이상 만든 뒤 첫 시작점 가까이 이동하고 시작점을 클릭한다 | 마지막 선이 첫 시작점에 정확히 연결되고 그리기가 종료된다 |
| ED-012B | 폐합 해제 | 닫힌 도형의 마지막 선을 삭제한다 | 열린 도형으로 돌아가며 연필 도구로 다시 이어 그릴 수 있다 |

| BX-005 | 면별 계산 | 면 1과 면 2에 길이가 다른 선을 입력한다 | 결과 패널의 면 1 폭과 면 2 높이가 독립적으로 계산된다 |
| BX-006 | 일반 전환 | 박스 작성 후 `일반`을 선택한다 | 면 1만 유지되고 면 2 편집 UI가 사라진다 |

## 5. 결함 기록 형식

결함을 발견하면 테스트 ID, 브라우저와 화면 크기, 입력 데이터, 실제 결과, 기대 결과, 재현 순서, 화면 캡처를 함께 기록한다. 계산 결함은 반드시 각 선의 입력 길이와 계산 길이도 같이 기록한다.
