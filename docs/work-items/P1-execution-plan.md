## 1. 목표

P1은 PostgreSQL·Prisma·Next.js 기반 위에 독자 인증과 조직 권한을 적용하고, 절곡 문서를 서버에 안전하게 저장하며, 계산·편집·전개·DXF를 실제 제작 검증이 가능한 수준으로 완성하는 단계다.

P1 완료 시 다음 수직 흐름이 연결돼야 한다.

```text
로그인
→ 절곡 초안 생성
→ 자동 저장
→ 새로고침 후 복원
→ 계산 preview
→ 템플릿 개정·게시
→ 제작 geometry·DXF
→ 감사 로그와 회귀 검증
```

## 2. 계승하는 확정 기준

- 입면도 화면·API·활성 모델·데이터 이전은 구현하지 않는다.
- 기계 연동은 메뉴·권한·`planned` 상태·계약 자리만 구현하고 실제 통신은 하지 않는다.
- MFC 로그인·실행 parameter·자격증명은 계승하지 않는다.
- 운영·개발·테스트 런타임은 PostgreSQL 서버 기반이며 SQLite를 사용하지 않는다.
- DB 변경은 Prisma Schema와 migration으로만 관리한다.
- MFC 결과를 무조건 복제하지 않고 오류·불합리성을 판정해 승인 기대값으로 검증한다.
- 한국어·대한민국 업무 환경을 1차 완료 범위로 한다.
- 사용자 본인이 모든 업무 결정과 검수를 담당한다.

## 5. 상세계획 작성 규칙

각 작업 착수 전에 `docs/work-items/P1-XX-*.md`를 만든다.

1. 사용자 결과와 포함·제외 범위를 확정한다.
2. MFC 근거, 레거시 DB와 현재 웹 코드를 연결한다.
3. Prisma·API·UI·권한·감사·운영 영향을 설계한다.
4. 정상·실패·충돌·보안 흐름을 작성한다.
5. migration은 빈 DB와 기존 DB upgrade를 시험한다.
6. 계산은 WEB-REFERENCE 또는 승인 개선 기대값으로 검증한다.
7. 자동 검증과 사용자 직접 검증을 분리한다.
8. 사용자 결정 게이트가 생기면 구현을 멈추고 확인받는다.
