## 1. P0 목표

P0의 목표는 기능을 많이 만드는 것이 아니라, 이후 구현을 되돌리게 만들 수 있는 범위·레거시·데이터·계산·DB 기반의 불확실성을 제거하고 PostgreSQL·Prisma 기반의 첫 수직 기능을 안전하게 구현할 수 있는 상태를 만드는 것이다.

P0 완료 시 다음 상태를 확보한다.

- 프로젝트 포함·제외 범위와 용어가 승인되어 있다.
- MFC의 대상 기능, 화면, 주요 코드와 사용자 검수자가 연결되어 있다.
- 실제 운영 데이터의 원본과 백업·복원 경로가 확인되어 있다.
- 레거시 테이블이 활성 이전, 보관 전용, 제외로 분류되어 있다.
- 외부 엔진과 라이선스의 확보 여부 및 대체 전략이 기록되어 있다.
- MFC 계산 golden master 20건, 차이 판정 기준과 측정 가능한 허용 오차가 승인되어 있다.
- 운영·로컬 PostgreSQL의 환경 기준이 확정되어 있다.
- Prisma Schema v1, migration, seed와 테스트 DB 초기화가 동작한다.
- Prisma transaction을 사용하는 서버 예제 경로가 통합 테스트를 통과한다.
- lint, typecheck, test, build와 migration 검사가 CI 품질 게이트에 포함된다.

## 2. 실행 원칙

1. 사용자 결정이 필요한 작업은 추정으로 확정하지 않는다.
2. 결정 전에도 가능한 읽기 전용 조사와 대안 비교는 진행한다.
3. 승인 게이트에 도달하면 의존 작업을 시작하기 전에 사용자에게 확인받는다.
4. 운영 DB, 원본 파일과 라이선스 자산은 승인 없이 변경·이동·공개하지 않는다.
5. 레거시 비밀번호와 자격증명 값은 문서·로그·저장소에 복사하지 않는다.
6. 입면도는 분석 근거와 보관 대상으로만 다루고 구현 모델에는 포함하지 않는다.
7. 기계 연동은 P0에서 현황과 의존성만 조사한다. 실제 Agent와 전송 코드는 구현하지 않는다.
8. DB 구현은 로컬 PostgreSQL과 Prisma를 사용하며 SQLite 우회 경로를 만들지 않는다.
9. 각 작업은 별도 작업 문서, 검증 증거와 완료 대조표를 남긴다.

### P0-03 운영 데이터 원본 확정

작업 문서: [P0-03 운영 데이터 원본 확정](./P0-03-operational-data-source.md)

세부 단계:

1. MFC DB 초기화·연결 코드를 읽어 SQLite와 서버 DB 분기 조건을 추적한다.
2. `hicom.ini` 등 설정 파일의 키 이름과 역할만 기록하고 secret 값은 노출하지 않는다.
3. `folddraw3.db`의 파일 메타데이터, 스키마와 row count를 읽기 전용으로 확인한다.
4. MFC 실행·인증·DB 연결이 웹 런타임에 필요하지 않음을 구분한다.
5. RDS 또는 운영 서버의 신규 PostgreSQL을 웹의 단일 원본으로 확정한다.
6. SQLite를 업무·테스트·다국어 런타임에서 제외한다.
7. 레거시 데이터는 필요 시 offline importer 입력으로만 사용한다.

게이트 `D0-03` 결과:

- MFC 실행 메커니즘과 운영 연결은 웹에서 계승하지 않음
- 웹 운영 단일 원본은 별도 PostgreSQL
- 개발·테스트는 로컬 PostgreSQL
- SQLite는 사용하지 않음
- 1차 언어는 한국어

### P0-04 레거시 스키마·키 매핑 — `DONE`

작업 문서: [P0-04 레거시 스키마·키 참고 매핑](./P0-04-legacy-schema-mapping.md)

6. Prisma 후보 엔터티와 `LegacyMapping` 전략을 연결한다.
7. 미확인 테이블을 실제 사용 여부 확인 목록으로 만든다.

차단 게이트 `D0-04`:

### P0-06 Golden master 1차 구축 — `DONE`

작업 문서: [P0-06 Golden master v1 구축](./P0-06-golden-master-v1.md)

세부 단계:

1. 각 타입·연신 옵션·박스·곡선·소수 경계를 포함하는 표본 매트릭스를 만든다.
2. 개인정보·거래처 식별정보의 비식별화 규칙을 정한다.

7. 재실행 절차와 결과 비교 형식을 정의한다.
8. 웹 테스트 fixture로 사용할 수 있는 중립 JSON 계약으로 변환한다.
9. MFC 값과 승인된 개선 기대값을 구분해 저장한다.
10. 표본 누락 영역과 100건 확대 계획을 기록한다.
