## 1. 목표

계산 정확도, geometry·DXF 비교, 성능, 가용성, 백업·복구, 보존 기간과 지원 환경을 자동 또는 수동으로 측정할 수 있는 기준으로 확정한다.

이 문서의 허용 오차는 세 가지를 구분한다.

1. **계산 판정 오차**: 동일 입력에 대한 서버 계산과 승인 기대값의 차이
2. **파일·geometry 오차**: 좌표 변환, DXF 왕복과 렌더링 과정에서 허용할 차이
3. **실제 제작 공차**: 소재·장비·작업 방식에 따라 정하는 현장 치수 허용 범위

계산 판정 오차와 실제 제작 공차를 섞지 않는다. 제작 공차 안에 들어온다는 이유로 계산 오류를 통과시키지 않으며, 계산값이 정확하더라도 현장 제작 공차 검증을 대신하지 않는다.

## 2. 승인된 선행 기준

- FIX 자동 보정은 소수를 보존한다.
- 최종 구간 길이에만 설정된 소수 자리와 처리 방식을 적용한다.
- 계산 제외 절곡의 기여는 절곡 양쪽 구간에서 모두 제외한다.
- RATIO 계산은 `angle < cutAngle`일 때만 적용한다.
- 브라우저 preview와 서버 승인 계산은 같은 버전의 규칙 패키지를 사용한다.
- 승인 계산은 서버에서 재실행하고 입력 hash, 엔진 버전, 규칙 버전과 함께 저장한다.
- PostgreSQL `numeric`과 Prisma `Decimal`을 서버의 공식 숫자 표현으로 사용한다.
- JavaScript `number` 결과만을 운영 승인값의 영속 형식으로 사용하지 않는다.

## 3. 정밀도와 비교 기준 제안

### 3.1 저장·계산 단위

| 항목 | 기준 단위 | 서버 저장 제안 | 화면 기본 표시 | 승인 비교 기준 |
|---|---|---|---|---|
| 길이·좌표·반경·컷 깊이·연신 | mm | `numeric(18,6)` | 설정값, 기본 0.1 mm | 같은 규칙 적용 후 정확히 일치 |
| 각도 | degree | `numeric(9,4)` | 0.01° | `0.01°` 이내 |
| 면적 | m² | `numeric(20,8)` | 0.0001 m² | `0.000001 m²` 이내 |
| 중량 | kg | `numeric(20,6)` | 0.001 kg | `0.001 kg` 이내 |
| 비율·할증률 | % 또는 무차원 | `numeric(18,8)` | 업무 필드별 정의 | 저장 scale 이내 |
| 단가·금액 | KRW | `numeric(18,2)` 계산 후 원 단위 확정 | 1원 | 최종 금액 `1원` 이내 |
| DXF 좌표 | mm | 내보내기 소수 6자리 | 해당 없음 | 좌표별 `0.01 mm` 이내 |

`numeric(p,s)`의 최종 크기는 P0-09 Prisma Schema review에서 실제 최대값과 함께 다시 확인하되, 이 단계에서 단위·반올림 위치는 고정한다.

### 3.2 Decimal 처리 순서

```text
문자열 입력 파싱
→ Prisma Decimal 또는 공용 Decimal 값으로 변환
→ 보정 중간값 소수 보존
→ 승인된 지점에서만 반올림·버림·올림
→ 서버 승인 계산
→ 문자열 Decimal과 단위로 응답·저장
```

- half 값의 반올림은 기존 승인 표본처럼 0에서 멀어지는 방향을 사용한다.
- `floor`와 `ceil`이라는 내부 이름은 음수 처리에서 수학 용어와 다르게 해석될 수 있으므로 P1-09에서 업무 이름과 실제 연산을 분리한다.
- 브라우저 숫자 입력은 표시 편의를 위한 값이며 서버가 다시 파싱하고 검증한다.
- 승인 snapshot의 Decimal은 JSON number가 아니라 정규화된 10진 문자열로 저장한다.
- 중간값에서 임의 반올림하지 않는다.

### 3.3 geometry·DXF와 제작 공차

| 구분 | 제안 기준 | 검증 방법 |
|---|---:|---|
| 동일 계산 엔진의 선별·최종 길이 | 승인 소수 규칙 적용 후 정확히 일치 | 단위·fixture 테스트 |
| 브라우저 preview와 서버 승인 결과 | Decimal 문자열 기준 정확히 일치 | 통합 테스트 |
| geometry 점·선 좌표 | `0.01 mm` 이내 | 좌표별 비교 |
| 닫힌 outline 시작·끝 거리 | `0.01 mm` 이내 | 폐합 거리 계산 |
| DXF export 후 재해석 좌표 | `0.01 mm` 이내 | export/import 왕복 테스트 |
| 각도 | `0.01°` 이내 | 절곡별 비교 |
| 면적 | `0.000001 m²` 이내 | Decimal 면적 비교 |
| 실제 제작 치수 | **사용자 결정 필요** | 캘리퍼스·줄자 등 현장 측정 절차 |

제작 공차 제안은 **완성 치수 기준 ±0.5 mm**이다. 재질·두께·절곡 수·장비에 따라 별도 공차표가 필요하면 이 값을 전역 기본값으로만 사용하고 재질 또는 공정 규칙으로 세분화한다.

## 4. 성능 목표 제안

성능 시험의 초기 부하 기준은 동시 로그인 50명, 동시 편집 20명이다. 실제 초기 사용 인원이 이보다 크면 승인 전에 부하 기준을 올린다.

| 대상 | 목표 | 측정 조건 |
|---|---:|---|
| 일반 조회 API | p95 `500 ms` 이하 | 캐시가 없어도 정상 인덱스를 사용하는 조회 |
| 일반 저장 API | p95 `800 ms` 이하 | 감사 로그와 개정 검증 포함 |
| 절곡 preview | p95 `200 ms` 이하 | 일반 도면 100개 구간 |
| 대형 preview | p95 `1 s` 이하 | geometry 객체 1,000개 |
| 자동 저장 | 마지막 변경 후 `2초` 이내 서버 반영 | 정상 네트워크, 충돌 없음 |
| 편집 초기 화면 | p95 `3 s` 이하 | 1,000개 객체 도면, 사내 표준 PC |
| 비동기 batch 요청 접수 | p95 `1 s` 이하 | 100개 작업, 결과 생성은 별도 SLA |
| API 오류율 | 월 `0.5%` 미만 | 검증 실패 4xx 제외 |
| 자동 저장 성공률 | 월 `99.9%` 이상 | 사용자가 취소한 요청 제외 |

측정은 로컬 개발 PC 결과가 아니라 운영과 유사한 PostgreSQL·컨테이너·네트워크 구성에서 최소 30분 동안 수행한다. p95는 최소 1,000회 요청을 기준으로 산출한다.

### 7.2 출력과 현장 프로그램

- 1단계 공식 출력은 서버 생성 PDF와 브라우저 인쇄다.
- 기본 용지는 A4이며 A3가 필요한 양식은 템플릿별로 명시한다.
- DXF는 단위를 mm로 고정하고 파일 version·layer·checksum을 기록한다.
- 특정 프린터 드라이버, CAD/CAM 제품과 NC 수신 프로그램은 실제 장비 목록을 확보한 뒤 호환성 표에 추가한다.
- 기계 연동은 1단계에서 메뉴와 `연동 예정` 상태만 검증한다.
